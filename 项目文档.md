# XF AI Agent 后端项目文档

## 项目概述

XF AI Agent 是一个基于 FastAPI、LangChain、LangGraph 构建的现代化智能代理后端系统。该系统提供了完整的用户管理、模型配置、对话处理、RAG（检索增强生成）等功能，支持多种大语言模型和向量数据库。

### 核心特性

- 🤖 **智能对话**: 基于 LangGraph 的多步骤推理和工具使用
- 📚 **RAG 支持**: 文档向量化检索，增强生成质量
- 🔐 **用户认证**: JWT 令牌认证，支持用户注册、登录、密码管理
- 🛠️ **模型管理**: 支持多种模型服务配置和动态切换
- 💾 **数据持久化**: MySQL 数据库存储，Redis 缓存支持
- 🔄 **流式输出**: 支持 Server-Sent Events (SSE) 实时响应
- 📊 **对话历史**: 完整的对话记录和管理功能
- 🧩 **MCP 集成**: 支持 Model Context Protocol 扩展

## 技术架构

### 技术栈

- **Web 框架**: FastAPI 0.116.0
- **异步运行时**: Uvicorn
- **AI 框架**: LangChain 0.3.27, LangGraph 0.6.2
- **数据库**: MySQL (PyMySQL), Redis
- **向量数据库**: FAISS
- **认证**: JWT (PyJWT), Passlib + Bcrypt
- **文档处理**: PyPDF
- **环境管理**: Python-dotenv
- **网络请求**: HTTPX
- **模板引擎**: Jinja2

## 核心模块详解

### 1. 应用入口 (main.py & app/main.py)

**main.py** - 应用启动入口
```python
import uvicorn
from app.main import app

if __name__ == "__main__":
    uvicorn.run(app, host="0.0.0.0", port=8000)
```

**app/main.py** - FastAPI 应用配置
- 配置 CORS 中间件
- 注册 API 路由
- 配置异常处理
- 设置应用元数据

### 2. 智能代理模块 (app/agent/)

**核心组件**:
- **agent.py**: LangGraph 代理的核心实现
- **graph.py**: 定义代理的图结构和工作流
- **nodes.py**: 实现各种处理节点（思考、工具调用、响应等）
- **state.py**: 管理代理状态和上下文
- **tools.py**: 定义代理可使用的工具集

**工作流程**:
1. 接收用户输入
2. 分析意图和上下文
3. 决定是否需要工具调用
4. 执行相应的处理节点
5. 生成最终响应

### 3. API 路由层 (app/api/v1/)

**主要接口**:

- **chat_api.py**: 对话相关接口
  - `POST /chat` - 发送消息并获取回复
  - `POST /chat/stream` - 流式对话接口

- **user_info_api.py**: 用户管理接口
  - `POST /users/register` - 用户注册
  - `POST /users/login` - 用户登录
  - `GET /users/profile` - 获取用户信息
  - `PUT /users/profile` - 更新用户信息
  - `POST /users/change-password` - 修改密码

- **model_setting_api.py**: 模型设置接口
  - `GET /model_setting/` - 获取模型服务列表
  - `POST /model_setting/` - 创建模型服务
  - `PUT /model_setting/{id}` - 更新模型服务
  - `DELETE /model_setting/{id}` - 删除模型服务

- **user_model_api.py**: 用户模型配置接口
  - `GET /user_model/` - 获取用户模型配置
  - `POST /user_model/` - 创建用户模型配置
  - `PUT /user_model/{id}/activate` - 激活模型配置

### 4. 数据模型层 (app/models/)

**核心模型**:

- **User**: 用户基础信息
- **UserModel**: 用户模型配置
- **ModelSetting**: 系统模型设置
- **ChatHistory**: 对话历史记录
- **UserMCP**: MCP 配置

### 5. 业务逻辑层 (app/services/)

**核心服务**:

- **ChatService**: 处理对话逻辑，集成 LangGraph 代理
- **RAGService**: 实现检索增强生成功能
- **DocumentService**: 处理文档上传和向量化
- **UserInfoService**: 用户信息管理
- **ModelSettingService**: 模型配置管理

### 6. 数据访问层 (app/db/)

基于 SQLAlchemy 的数据访问层，提供统一的 CRUD 操作接口。

**特点**:
- 统一的基础 CRUD 操作
- 事务管理
- 连接池管理
- 查询优化

## 核心功能实现

### 1. 用户认证系统

**认证流程**:
1. 用户注册/登录
2. 生成 JWT 令牌
3. 令牌验证中间件
4. 权限控制

**安全特性**:
- 密码 Bcrypt 加密
- JWT 令牌过期控制
- 请求频率限制
- CORS 安全配置

### 2. 模型管理系统

**模型分类**:
- **对话模型** (chat): GPT、Claude、Gemini 等
- **嵌入模型** (embedding): text-embedding、BGE 等
- **视觉模型** (vision): GPT-4V、Claude-3-Vision 等
- **其他模型** (other): 特殊用途模型

**配置管理**:
- 系统预定义模型服务
- 用户自定义模型配置
- 动态模型切换
- 多模型并行支持

### 3. RAG 检索系统

**实现架构**:
```
文档上传 → 文本提取 → 分块处理 → 向量化 → FAISS存储
                                                    ↓
用户查询 → 向量化 → 相似度检索 → 上下文注入 → LLM生成
```

**核心组件**:
- 文档解析器 (PDF/TXT)
- 文本分块器
- 向量嵌入模型
- FAISS 向量数据库
- 相似度检索器

### 4. 流式对话系统

**技术实现**:
- Server-Sent Events (SSE)
- 异步流式处理
- 实时响应推送
- 连接状态管理

**消息格式**:
```
data: {"type": "token", "content": "Hello"}
data: {"type": "thinking", "content": "思考过程..."}
data: {"type": "end", "content": null}
```

## 数据库设计

### 核心表结构

**用户表 (users)**
```sql
CREATE TABLE users (
    id INT PRIMARY KEY AUTO_INCREMENT,
    username VARCHAR(50) UNIQUE NOT NULL,
    email VARCHAR(100) UNIQUE NOT NULL,
    password_hash VARCHAR(255) NOT NULL,
    full_name VARCHAR(100),
    is_active BOOLEAN DEFAULT TRUE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);
```

**模型设置表 (model_settings)**
```sql
CREATE TABLE model_settings (
    id INT PRIMARY KEY AUTO_INCREMENT,
    service_name VARCHAR(100) NOT NULL,
    service_type VARCHAR(50) NOT NULL,
    service_url VARCHAR(255) NOT NULL,
    api_key_template VARCHAR(255),
    icon VARCHAR(50) DEFAULT 'FiCpu',
    models JSON NOT NULL,  -- 分类存储: {"chat": [], "embedding": [], "vision": [], "other": []}
    description TEXT,
    is_system_default BOOLEAN DEFAULT FALSE,
    is_enabled BOOLEAN DEFAULT TRUE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);
```

**用户模型配置表 (user_models)**
```sql
CREATE TABLE user_models (
    id INT PRIMARY KEY AUTO_INCREMENT,
    user_id INT NOT NULL,
    model_setting_id INT,
    service_name VARCHAR(100) NOT NULL,
    selected_model VARCHAR(100) NOT NULL,
    api_key VARCHAR(255),
    api_url VARCHAR(255),
    custom_config JSON,
    is_active BOOLEAN DEFAULT FALSE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE,
    FOREIGN KEY (model_setting_id) REFERENCES model_settings(id) ON DELETE SET NULL
);
```

**对话历史表 (chat_histories)**
```sql
CREATE TABLE chat_histories (
    id INT PRIMARY KEY AUTO_INCREMENT,
    user_id INT NOT NULL,
    session_id VARCHAR(100) NOT NULL,
    title VARCHAR(255),
    messages JSON NOT NULL,
    model_info JSON,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE
);
```

## 部署指南

### 1. 开发环境部署

```bash
# 1. 克隆项目
git clone <repository-url>
cd xf-ai-agent

# 2. 创建虚拟环境
uv venv
source .venv/bin/activate  # Linux/Mac
# .venv\Scripts\activate   # Windows

# 3. 安装依赖
uv pip install .

# 4. 配置环境变量
cp .env.example .env
# 编辑 .env 文件

# 5. 初始化数据库
mysql -u root -p < mysql.sql

# 6. 启动应用
python main.py
```

### 2. 生产环境部署

**使用 Docker**:
```dockerfile
FROM python:3.12-slim

WORKDIR /app
COPY . .

RUN pip install uv
RUN uv pip install .

EXPOSE 8000
CMD ["uvicorn", "main:app", "--host", "0.0.0.0", "--port", "8000"]
```

**使用 Nginx + Gunicorn**:
```bash
# 安装 Gunicorn
uv pip install gunicorn

# 启动应用
gunicorn main:app -w 4 -k uvicorn.workers.UvicornWorker --bind 0.0.0.0:8000
```
