# 06 前端流程卷轴与页面终端

## 1. 前端不只是聊天窗口

这个项目的前端承担三类能力：

1. 对话输入输出
2. 流程卷轴可视化
3. 页面终端与运行时面板

## 2. 流程卷轴的目标

流程卷轴不是简单展示日志，而是要让用户一眼看懂：

- 老板发了什么任务
- 掌柜怎么判断
- 是否有司员介入
- 是否需要审批
- 最终如何回禀

## 3. 运行时面板

当前卷轴已经补齐了 Runtime 相关面板：

- Tool Call
- Artifact
- Exec
- Memory
- Context
- Hook

这些面板让“模型之外发生了什么”变得可见。

## 4. 页面终端

页面终端的目标是：

- 在页面里像 Codex 一样执行命令
- 不打开系统 Terminal
- 和聊天会话绑定
- 和工作目录绑定
- 左侧直接浏览项目目录树
- 右侧直接打开、编辑、保存文本文件

### 4.1 页面终端显示什么

- 当前工作目录
- 工作目录树
- 当前文件与编辑区
- 要执行的命令
- 命令输出
- 退出码
- 运行状态

### 4.2 页面终端实际怎么执行

页面终端不是浏览器本地执行，也不是弹出系统命令行窗口。

真实执行位置取决于后端服务运行在哪里：

- 如果后端跑在你的本机，就在你本机执行；
- 如果后端部署到服务器，就在服务器执行。

执行链路是：

1. 前端先把当前聊天会话绑定到一个工作目录；
2. 后端校验该目录是否位于允许的工作区根目录内；
3. 前端通过目录树接口和文件接口展示项目内容；
4. 用户在页面中输入命令或直接编辑文件；
5. 后端以受控子进程的方式在绑定目录或其子目录中运行命令；
6. 前端轮询命令状态并展示输出。

因此，页面终端更准确地说是“页面工作台 + 后端受控执行器”。

### 4.3 页面终端不做什么

- 不允许任意目录运行
- 不默认把自然语言当成命令执行
- 不把工作目录之外的文件暴露给页面
- 不绕过后端控制

### 4.4 当前工作台模式

当前工作台已经具备这些能力：

- 绑定目录
- 浏览目录树
- 新建文件
- 新建目录
- 重命名文件或目录
- 删除文件或目录
- 打开文本文件
- 编辑并保存文本文件
- 在所选目录执行常见开发命令
- 全屏展开为更接近 Codex 的工作台视图

当前更适合的命令包括：

- `python3 app.py`
- `pytest`
- `uv run python -m unittest`
- `javac Main.java`
- `java Main`
- `npm run dev`
- `git status`

当前还不是完整的 PTY 交互式 shell。后续如果要做得更像 Codex 的原生终端，再升级成 `WebSocket + PTY` 会更合适。

## 5. 为什么要做成页面内终端

因为这个项目不是纯工程师工具，而是一个“流程可视化产品”。

把终端能力也放到页面里，能让老板模式和工位模式共存。
