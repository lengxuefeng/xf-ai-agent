# 07 数据库与会话状态

## 1. 数据库在这个项目中的角色

数据库不只是保存用户和聊天记录，还承担：

- 会话历史
- 审批记录
- 模型配置
- RAG 文档与向量数据

## 2. 会话状态的核心对象

### 聊天会话

保存：

- 会话 ID
- 标题
- 创建时间
- 更新时间

### 聊天消息

保存：

- 用户内容
- 模型内容
- thinking trace
- workflow trace
- runtime snapshot
- runtime artifacts

### Interrupt 审批记录

保存：

- session_id
- message_id
- 审批状态
- action_name
- action_args
- checkpoint 信息

## 3. 为什么 `extra_data` 很重要

当前消息表里的 `extra_data` 是前后端流程回放的关键：

- 刷新页面后仍能回看流程
- 前端不只显示正文，还能展示运行时状态

## 4. 还需要重点理解的点

### 4.1 运行时快照

它描述“这一轮执行走到哪里了”。

### 4.2 产物清单

它描述“这轮执行生成了哪些文件/记录”。

### 4.3 审批记录

它决定中断能否被精准恢复。
