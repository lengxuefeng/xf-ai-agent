# 02 系统架构

## 1. 总体架构

系统可以分成四层：

1. **前端交互层**
2. **API / SSE 接口层**
3. **多 Agent 编排层**
4. **Harness Runtime 层**

**支撑层**:

- **通用模块层**: 提供通用工具和辅助功能
- **配置管理层**: 管理系统配置
- **数据库层**: 数据持久化
- **核心模块层**: 提供核心功能
- **工具函数层**: 提供通用工具函数

## 2. 主流程

```text
用户请求
-> Chat API
-> 前置规则拦截（Zero-LLM）
-> Supervisor Graph
-> Agent 调度
-> Runtime 能力调用
-> SSE 流式返回
-> 前端对话区 + 流程卷轴 + 页面终端
```

## 3. 编排层职责

编排层主要由以下部分构成：

- **Supervisor 编排图**: 负责任务分发和协调
- **专业 Agent**: 代码、搜索、SQL、天气、医疗、云游等专业化 Agent
- **GraphRunner**: 图执行器，使用生产者-消费者异步模型驱动图执行
- **前置规则拦截**: Zero-LLM 快速响应，避免不必要的图执行
- **Agent 注册**: 统一的 Agent 注册和管理机制
- **规则支持**: Supervisor 规则支持和辅助函数

这部分决定的是"谁来做"和"怎么组织任务"。

### 3.1 Supervisor 编排图

- 负责任务分发和协调
- 根据用户问题决定调用哪个专业 Agent
- 支持简单直答、单 Agent、多任务拆解等模式
- **supervisor.py**: Supervisor 编排图实现
- **supervisor_rule_support.py**: Supervisor 规则支持
- **supervisor_support.py**: Supervisor 辅助函数

### 3.2 GraphRunner 执行器

- 使用生产者-消费者异步模型
- 图执行在后台线程，主线程消费队列推 SSE
- 支持前置规则拦截
- 支持审批中断和恢复
- 支持运行时观测
- **graph_runner.py**: 图执行器核心实现（115KB，约 3000+ 行）

## 4. Runtime 层职责

Runtime 层包含：

- **Core**: RunContext、SessionManager、WorkflowEventBus
- **Workspace**: 文件系统和工作区管理
- **Exec Runtime**: Bash / Python 受控执行
- **Memory**: 记忆管理和 AGENTS.md 注入
- **Tool Runtime**: Web Search / MCP / Tool Registry
- **Context Builder**: 上下文构建和注入
- **Hook Manager**: Hooks / Approval / Observability
- **Prompt Registry**: 系统提示词管理

这部分决定的是"Agent 怎么真正干活"。

### 4.1 Runtime 核心

- **RunContext**: 单次运行的统一上下文管理
- **SessionManager**: 运行时会话管理和状态追踪
- **WorkflowEventBus**: 工作流事件总线和持久化
- **LiveStreamBus**: 子 Agent 实时流事件总线

### 4.2 Workspace 管理

- 为每个会话、每次运行准备工作区
- 保存中间产物和最终响应
- 记录 workflow trace
- 支持 Path Guard 安全约束

### 4.3 受控执行

- 限制命令白名单
- 限制工作目录（Path Guard）
- 返回 stdout / stderr / exit code
- 支持会话管理

### 4.4 记忆管理

- 注入项目记忆（AGENTS.md）
- 注入协作规则
- 注入用户偏好
- 支持结构化记忆片段

### 4.5 工具注册

- 统一的工具注册机制
- 支持 MCP（Model Context Protocol）
- 支持外部工具集成
- 工具执行和结果处理

### 4.6 上下文工程

- 裁剪历史
- 注入记忆
- 注入 RAG 片段
- 控制 token 预算

### 4.7 Hooks 机制

- 审批 Hook
- 质量门禁
- 工具守卫
- 事件追踪
- 支持自定义 Hook

## 5. 通用模块层职责

通用模块层主要由以下部分构成：

- **HTTP 工具**: 提供通用的 HTTP 客户端
- **扩展支持**: 为外部 API 调用提供支持

这部分提供通用的工具和辅助功能。

### 5.1 HTTP 工具

- **HTTP 客户端**: 通用的 HTTP 客户端，用于外部 API 调用
- **配置管理**: HTTP 请求的配置管理
- **错误处理**: HTTP 请求的错误处理

### 5.2 扩展支持

- **可扩展架构**: 支持添加新的通用工具
- **统一接口**: 提供统一的工具接口
- **模块化设计**: 工具模块化，易于维护

## 6. 配置管理层职责

配置管理层主要由以下部分构成：

- **关键词配置**: 定义各种关键词和短语
- **运行时配置**: 系统运行时的各种参数
- **规则配置**: 前置拦截规则配置
- **测试配置**: 测试环境配置

这部分管理系统配置。

### 6.1 关键词配置

- **关键词定义**: 定义各种业务关键词
- **关键词匹配**: 关键词匹配逻辑
- **关键词管理**: 关键词的增删改查

### 6.2 运行时配置

- **系统参数**: 系统运行时的各种参数
- **性能调优**: 性能相关参数
- **安全配置**: 安全相关配置

### 6.3 规则配置

- **拦截规则**: 前置拦截规则定义
- **规则文件**: YAML 格式的规则文件
- **规则加载**: 规则的加载和解析

## 7. 前端职责

前端承担三类展示：

1. **聊天消息**: 用户与 Agent 的交互界面
2. **流程卷轴**: 可视化展示"老板发令 -> 掌柜调度 -> 司员执行 -> 汇总回禀"全过程
3. **页面终端与运行时面板**: 受控命令执行界面和运行状态监控

它不是纯展示层，而是部分承接了"可观测性产品化"的角色。

## 6. 技术特点

### 6.1 异步流式处理

- GraphRunner 使用生产者-消费者模型
- 图执行在后台线程，主线程消费队列推 SSE
- 全程不阻塞事件循环，支持真正的并发流式推送

### 6.2 前置规则拦截

- Zero-LLM 快速响应
- 避免不必要的图执行
- 支持多规则复合拦截
- 意图覆盖率防爆盾

### 6.3 审批中断恢复

- 支持 Interrupt 挂起
- 支持审批恢复
- 支持状态回填
- 完整的审批工作流

### 6.4 运行时观测

- Workflow 事件流
- 日志流
- 状态快照
- 实时监控和回放

### 6.5 Session 管理

- 运行时会话管理
- 状态追踪
- Session Pool 预热机制
- 性能优化

## 7. 与 Harness-first 架构的关系

你当前项目的最佳定位是：

`强编排 + 完整的 Harness Runtime + 通用模块 + 配置管理`

不是推翻原来的编排系统，而是在现有编排系统下面补一层更标准的 Runtime，并且已经形成了完整的 Harness Runtime 实现体系。

新增的通用模块层和配置管理层进一步增强了系统的可扩展性和可配置性。
