# 02 系统架构

## 1. 总体架构

系统可以分成四层：

1. 前端交互层
2. API / SSE 接口层
3. 多 Agent 编排层
4. Harness Runtime 层

## 2. 主流程

```text
用户请求
-> Chat API
-> Supervisor Graph
-> Domain Router
-> Intent Router
-> (简单直答 / 单 Agent / 多任务拆解)
-> Runtime 能力调用
-> SSE 流式返回
-> 前端对话区 + 流程卷轴 + 页面终端
```

## 3. 编排层职责

编排层主要由以下部分构成：

- `Domain Router`
- `Intent Router`
- `Parent Planner`
- `Dispatcher`
- `Worker`
- `Aggregator`

这部分决定的是“谁来做”和“怎么组织任务”。

## 4. Runtime 层职责

Runtime 层包含：

- Workspace
- Exec Runtime
- Memory
- Tool Runtime
- Context Builder
- Hook Manager
- Prompt Registry

这部分决定的是“Agent 怎么真正干活”。

## 5. 前端职责

前端承担三类展示：

1. 聊天消息
2. 流程卷轴
3. 页面终端与运行时面板

它不是纯展示层，而是部分承接了“可观测性产品化”的角色。

## 6. 与 Harness-first 架构的关系

你当前项目的最佳定位是：

`强编排 + 正在补齐 Harness`

不是推翻原来的编排系统，而是在现有编排系统下面补一层更标准的 Runtime。
