# 04 多 Agent 编排

## 1. 为什么要多 Agent

多 Agent 的好处不是“看起来高级”，而是为了把职责拆清楚。

例如：

- 普通聊天归 `CHAT`
- 代码问题归 `code_agent`
- 天气归 `weather_agent`
- 医疗或业务归对应专业 Agent

## 2. 编排核心节点

### Domain Router

判断问题属于哪个大的业务域。

### Intent Router

在业务域内进一步判断该交给哪个具体 Agent。

### Parent Planner

遇到复杂请求时，把任务拆成多个子任务。

### Dispatcher / Worker

把子任务分发给具体执行者。

### Aggregator

把多个结果整理成最终回禀。

## 3. 直办和拆分的区别

### 掌柜直办

适用于：

- 简单问答
- 普通咨询
- 不需要工具或外部系统的场景

### 司员执行

适用于：

- 需要专业工具
- 需要联网
- 需要代码执行
- 需要多步骤完成

## 4. 当前项目的优势

这套编排非常适合你的产品形态，因为前端流程卷轴正好需要明确看到：

- 老板发令
- 掌柜分派
- 司员执行
- 总管回禀
