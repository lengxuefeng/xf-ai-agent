# 09 测试与回归

## 1. 为什么要分层测试

这个项目的风险点不只在模型输出，还在：

- **路由是否正确**: Supervisor 能否正确判断问题类型
- **审批是否稳定**: 审批流程是否稳定可靠
- **恢复是否成功**: 中断状态能否正确恢复
- **命令是否越界**: 命令执行是否遵守安全边界
- **前端流程是否和后端事件对齐**: 流程卷轴展示是否准确
- **Runtime 能力是否正常**: Workspace、Exec、Memory 等能力是否正常工作

所以测试必须分层，每个层次都要覆盖。

## 5. 关键业务用例

### 5.1 必须回归的典型场景

#### 场景 1: 简单对话

**输入**: `java能学吗`

**预期结果**:

- 前置规则拦截或 Supervisor 直接回答
- 流程卷轴展示简单执行流程
- 返回学习建议

#### 场景 2: 代码生成

**输入**: `帮我写一个 Python hello world`

**预期结果**:

- 调用 CodeAgent
- 生成代码
- 不进入审批流程
- 返回代码结果

#### 场景 3: 代码执行

**输入**: `帮我执行这段 Python 代码`

**预期结果**:

- 调用 CodeAgent
- 识别为执行意图
- 触发审批流程
- 用户审批后执行
- 返回执行结果

#### 场景 4: 天气查询

**输入**: `查一下上海天气`

**预期结果**:

- 调用 WeatherAgent
- 查询天气
- 返回天气结果

#### 场景 5: 需要人工审批的工具调用

**输入**: 需要调用高风险工具的请求

**预期结果**:

- Agent 判断需要调用高风险工具
- 触发审批流程
- 前端展示审批界面
- 用户审批后执行
- 返回执行结果

#### 场景 6: 多任务拆分

**输入**: 复杂的多任务请求

**预期结果**:

- Supervisor 拆分任务
- 调用多个 Agent
- 并行或串行执行
- 汇总结果
- 返回最终结果

### 5.2 边界测试用例

#### 边界 1: 超长输入

**输入**: 超长文本输入

**预期结果**:

- 不触发规则拦截
- 正常进入图执行
- 上下文裁剪
- 正常返回结果

#### 边界 2: 空输入

**输入**: 空字符串或只有空格

**预期结果**:

- 返回错误提示
- 不进入图执行

#### 边界 3: 非法字符

**输入**: 包含特殊字符的输入

**预期结果**:

- 正常处理或返回错误提示
- 不导致系统崩溃

**预期结果**:

### 6.2 异常路径

#### 测试 1: 绑定非法目录

**步骤**:

1. 尝试绑定非法目录
2. 验证绑定失败

**预期结果**:

- 绑定失败
- 显示错误提示
- 不允许越界

#### 测试 2: 命令超出白名单

**步骤**:

1. 尝试执行超出白名单的命令
2. 验证命令被拒绝

**预期结果**:

- 命令被拒绝
- 显示错误提示
- 不执行命令

#### 测试 3: 命令运行超时

**步骤**:

1. 执行会超时的命令
2. 等待超时
3. 验证超时处理

**预期结果**:

- 命令被终止
- 显示超时提示
- 返回超时错误

#### 测试 4: 停止命令

**步骤**:

1. 启动长时间运行的命令
2. 停止命令
3. 验证命令被停止

**预期结果**:

- 命令被停止
- 显示停止提示
- 释放资源
