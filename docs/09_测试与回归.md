# 09 测试与回归

## 1. 为什么要分层测试

这个项目的风险点不只在模型输出，还在：

- 路由是否正确
- 审批是否稳定
- 恢复是否成功
- 命令是否越界
- 前端流程是否和后端事件对齐

所以测试必须分层。

## 2. 后端测试

建议至少覆盖：

- Supervisor 路由
- Runtime Harness
- Chat Stream 稳定性
- Interrupt 审批恢复
- Terminal 目录边界

本次回归用到的典型命令：

```bash
cd xf-ai-agent
PYTHONPATH=app uv run python -m unittest tests.test_runtime_harness_layers tests.test_supervisor_multitask_split tests.test_chat_stream_stability -v
```

## 3. 前端测试

建议至少覆盖：

- `npm run build`
- 关键组件 lint
- 流程卷轴展示
- 页面终端展示
- 工作目录绑定

## 4. 关键业务用例

必须回归的典型场景：

- `java能学吗`
- `帮我写一个 Python hello world`
- `帮我执行这段代码`
- `查一下上海天气`
- `需要人工审批的工具调用`

## 5. 页面终端的专项测试

### 正常路径

- 绑定合法目录
- 执行合法命令
- 输出正常回显

### 异常路径

- 绑定非法目录
- 命令超出白名单
- 命令运行超时
- 停止命令

## 6. 测试结论记录建议

每次较大改动后，至少记录：

- 测试命令
- 是否通过
- 已知残留问题
- 是否影响生产
