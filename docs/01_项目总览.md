# 01 项目总览

## 1. 这是什么项目

XF-AI-Agent 是一个“多 Agent 编排 + Harness Runtime + 前端流程可视化”的完整系统。

它既不是只有聊天窗口的 AI 应用，也不是只会跑代码的 Agent，而是一个兼顾以下目标的产品：

- 能对话
- 能路由
- 能拆任务
- 能调用工具
- 能审批
- 能回放流程
- 能在页面里进行受控执行

## 2. 解决什么问题

裸模型通常有几个硬伤：

1. 不会稳定记住项目上下文
2. 不会直接操作真实工作区
3. 不会安全地执行命令
4. 不会把整个执行过程结构化暴露给用户

这个项目就是围绕这些问题构建的。

## 3. 两层核心结构

### 3.1 编排层

编排层关注：

- 这句话属于哪个业务域
- 应该交给哪个 Agent
- 是否要拆成多个任务
- 最后怎么汇总回老板

### 3.2 运行时层

运行时层关注：

- 工作目录和文件产物
- 代码或命令如何安全执行
- 记忆如何注入
- 工具如何统一调用
- 审批如何挂起与恢复
- 运行轨迹如何被前端展示

## 4. 前端产品形态

前端不是只负责显示聊天消息，而是承担两种角色：

### 4.1 老板模式

让用户看到：

- 老板发了什么任务
- 掌柜如何调度
- 司员执行到了哪里
- 最终怎么回禀

### 4.2 工位模式

让用户看到：

- 当前工作目录
- 页面终端命令
- 输出流
- 命令状态
- 文件产物

## 5. 当前项目的关键价值

### 5.1 对业务型 Agent 系统友好

如果你的目标是做企业内部工作流、咨询助手、报告助手、审批助手，这套架构非常适合。

### 5.2 对“老板可观测”友好

相比很多只显示最终答案的 Agent，这个项目更强调“过程可见、责任可见、风险可见”。

### 5.3 适合继续演进成平台

它现在已经不仅是单体功能，更像一个可以继续平台化的 Agent Operating Layer。

## 6. 你应该怎么理解这个项目

最容易理解的方式是：

`上层是掌柜调度系统`

`下层是 Agent 的工作底座`

二者结合起来，才是完整的生产级 Agent 系统。
