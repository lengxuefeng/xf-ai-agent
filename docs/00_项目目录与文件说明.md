```

│   ├── api/                  # API 接口层
│   ├── common/               # 通用模块（新增）
│   ├── config/               # 配置管理
│   ├── constants/            # 常量定义
│   ├── core/                 # 核心模块
│   ├── db/                   # 数据库操作
│   ├── exceptions/           # 异常处理
│   ├── logs/                 # 日志目录
│   ├── models/               # 数据模型
│   ├── runtime/              # Harness Runtime
│   ├── schemas/              # Pydantic 模型
│   ├── services/             # 业务服务层
│   └── utils/                # 工具函数
├── docs/                     # 文档目录

## 1. 应用主目录（app/）

### 1.1 Agent 编排层（app/agent/）

**作用**: 多 Agent 编排和任务调度

**目录结构**:
```
app/agent/
├── __init__.py              # 包初始化文件
├── base.py                  # Agent 基类
├── graph_runner.py          # 图执行器（核心文件）
├── graph_runner_events.py   # 图执行事件定义
├── graph_state.py           # 图状态定义
├── registry.py              # Agent 注册表
├── agents/                  # 专业 Agent 目录
│   ├── __init__.py
│   ├── code_agent.py        # 代码 Agent
│   ├── medical_agent.py     # 医疗 Agent
│   ├── search_agent.py      # 搜索 Agent
│   ├── sql_agent.py         # SQL Agent
│   ├── weather_agent.py     # 天气 Agent
│   └── yunyou_agent.py      # 云游 Agent
├── graphs/                  # 编排图目录
│   ├── __init__.py
│   ├── checkpointer.py      # LangGraph Checkpointer
│   ├── state.py             # 图状态定义
│   ├── supervisor.py        # Supervisor 编排图
│   ├── supervisor_rule_support.py   # Supervisor 规则支持
│   └── supervisor_support.py       # Supervisor 辅助函数
├── gateway/                 # 网关目录
│   ├── __init__.py
│   └── federated_query_gateway.py  # 联邦查询网关
├── llm/                     # LLM 模型管理
│   ├── __init__.py
│   ├── loader_llm_multi.py  # 多 LLM 加载器
│   ├── model_config.py      # 模型配置
│   ├── ollama_model.py      # Ollama 模型
│   └── unified_loader.py    # 统一加载器
├── middleware/              # 中间件
│   ├── __init__.py
│   └── common_middleware.py # 通用中间件
├── policy/                  # 策略引擎
│   ├── __init__.py
│   └── sql_policy_engine.py # SQL 策略引擎
├── prompts/                 # 提示词管理
│   ├── __init__.py
│   ├── code_prompt.py       # 代码提示词
│   ├── medical_prompt.py    # 医疗提示词
│   ├── sql_prompt.py        # SQL 提示词
│   ├── supervisor_prompt.py # Supervisor 提示词
│   ├── tools_prompt.py      # 工具提示词
│   └── yunyou_prompt.py     # 云游提示词
├── rag/                     # RAG 模块
│   ├── __init__.py
│   └── vector_store.py      # 向量存储
├── rules/                   # 规则管理
│   ├── __init__.py
│   ├── actions.py           # 规则动作
│   └── registry.py          # 规则注册表
├── template/                # 模板管理
│   └── __init__.py
├── text_to_sql/             # 文本转 SQL
│   └── __init__.py
└── tools/                   # Agent 工具
    ├── __init__.py
    ├── retriever.py         # 检索器
    ├── search_tools.py      # 搜索工具
    ├── sql_tools.py         # SQL 工具
    ├── weather_tools.py     # 天气工具
    └── yunyou_tools.py      # 云游工具
```

**核心文件说明**:

- **base.py**: 定义了所有 Agent 的基类，提供 Agent 的通用接口
- **graph_runner.py**: 图执行器的核心实现，负责驱动 Supervisor 图执行并产生 SSE 流
- **graph_runner_events.py**: 定义了图执行过程中的事件类型和结构
- **graph_state.py**: 定义了图执行过程中的状态结构
- **registry.py**: Agent 注册表，管理所有可用的 Agent

### 1.2 API 接口层（app/api/）

**作用**: 提供 HTTP API 接口

**目录结构**:
```

├── __init__.py              # 包初始化文件
└── v1/                      # API v1 版本
    ├── __init__.py
    ├── chat_api.py          # 聊天 API

## 7. 模块依赖关系

### 7.1 依赖层次

```
app/
├── core/           # 最底层，不依赖其他业务模块
├── utils/          # 工具层，被其他模块依赖
├── runtime/        # Runtime 层，依赖 core 和 utils
├── db/             # 数据库层，依赖 core
├── models/         # 模型层，依赖 core
├── schemas/        # Schema 层，依赖 models
├── services/       # 服务层，依赖 runtime、db、models
├── agent/          # Agent 层，依赖 services、runtime
├── api/            # API 层，依赖 services、agent
└── config/         # 配置层，被所有模块依赖
```

### 7.2 主要依赖关系

- **Agent → Services**: Agent 层依赖服务层
- **Services → Runtime**: 服务层依赖 Runtime 层
- **API → Agent**: API 层依赖 Agent 层
- **Runtime → Core**: Runtime 层依赖核心模块
- **All → Config**: 所有模块依赖配置层

## 9. 总结

这个项目的目录结构清晰，职责分明：

- **app/agent/**: Agent 编排和任务调度
