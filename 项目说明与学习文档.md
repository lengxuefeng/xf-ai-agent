# XF-AI-Agent 项目说明与学习文档

## 1. 项目概述

XF-AI-Agent 是一套基于 **LangChain + LangGraph + FastAPI** 的多智能体 AI 对话系统，包含：

| 项目 | 技术栈 | 说明 |
|------|--------|------|
| `xf-ai-agent`（后端） | Python / FastAPI / LangGraph / PostgreSQL | 多 Agent 协同，支持 Supervisor 路由、RAG、Interrupt 人工审批 |
| `xf-ai-agent-app`（前端） | React / Vite / CSS | 聊天 UI、模型选择、SSE 流式渲染、主题切换 |

---

## 2. 系统架构

```
┌─────────────┐    SSE / REST     ┌──────────────────────────────────────┐
│  React 前端  │ ◄──────────────► │  FastAPI 后端                         │
│  (Vite)     │                  │                                      │
└─────────────┘                  │  main.py (应用入口)                    │
                                 │    ├─ api/v1/  (路由)                  │
                                 │    ├─ services/chat_service.py (业务层) │
                                 │    ├─ agent/graph_runner.py (调度中枢)  │
                                 │    │    └─ graphs/supervisor.py (路由)  │
                                 │    │         ├─ code_agent (代码)       │
                                 │    │         ├─ medical_agent (医疗)    │
                                 │    │         ├─ sql_agent (SQL)        │
                                 │    │         └─ yunyou_agent (云游)     │
                                 │    ├─ agent/rag/ (RAG 知识库)          │
                                 │    └─ db/ (PostgreSQL)                │
                                 └──────────────────────────────────────┘
```

### 核心调用链

```
用户输入 → ChatService.chat_stream()
         → GraphRunner.stream_run()
         → Supervisor 路由决策（关键词 → LLM 语义 → 规则兜底）
         → 子 Agent 执行（如 YunyouAgent）
         → SSE 事件流推送至前端
```

## 5. LangChain / LangGraph 核心概念对照

| 概念 | 本项目对应 | 说明 |
|------|-----------|------|
| **Graph** | `supervisor.py` → `create_graph()` | LangGraph 编译图，定义节点和边 |
| **State** | `state.py` → `GraphState` | 图的状态类型，包含 `messages`、`next`、`session_id` 等字段 |
| **Node** | Supervisor 节点 + 各 Agent 节点 | 图中的处理单元 |
| **Edge** | `supervisor.py` 中的 `add_conditional_edges` | 基于路由决策的条件跳转 |
| **Checkpointer** | `PostgresSaver` | 会话状态持久化到 PostgreSQL |
| **Interrupt** | `yunyou_agent.py` 中的 `interrupt()` | 暂停执行等待人工审批，前端显示审批按钮 |
| **Command** | `GraphRunner._handle_resume()` 中的 `Command(resume=...)` | 恢复被中断的执行流 |
| **ChatModel** | `UnifiedModelLoader.load_chat_model()` | 支持 OpenAI、Gemini、Ollama、通义等多种模型 |
| **Tool** | 各 Agent 中的 `@tool` 装饰器函数 | 扩展 Agent 能力（如 SQL 查询、代码执行等） |

## 6. Supervisor 路由决策流程

Supervisor 是整个系统的"大脑"，负责将用户请求路由到合适的子 Agent：

```
用户输入
  │
  ▼
[1] 关键词匹配（registry.py 中的 keywords）
  │ 命中 → 直接路由
  │ 未命中 ↓
  ▼
[2] LLM 语义分析（RouteDecision Pydantic 模型）
  │ 置信度 ≥ 0.7 → 路由到选中 Agent
  │ 置信度 < 0.7 ↓
  ▼
[3] 规则兜底 → 路由到默认 Agent（chat_agent）
```

## 7. SSE（Server-Sent Events）实时流式通信架构

为了实现**真正的全栈实时流式输出**（不仅包含大模型 Token 的逐字打印，还包括工具加载、API 请求、底层报错等 Python 系统级日志的实时展示），本项目对 `GraphRunner` 进行了深度定制：

```
                    ┌──────────────────────────────────────────────┐
                    │               GraphRunner.stream_run()       │
                    │                                              │
 前端 api.stream() │  [主线程 loop]         [后台线程 Graph Worker] │
     │              │     │ 阻塞取数据            │                │
     │◄── POST ─────┼─────┼───────────────────────┤ graph.stream() │
     │              │     │                       │                │
     │◄── thinking ─┼─────┼◄─────── 队列 ◄────────┤ 产生日志/节点事件│
     │◄── thinking ─┼─────┼◄──(event_queue.get)───┤ (log.info)     │
     │◄── stream ───┼─────┼◄─────── 队列 ◄────────┤ (AIMessage)    │
     │              │     │                       │                │
     │◄── error ────┼─────┼◄─────── 队列 ◄────────┤ 抛出异常        │
                    └──────────────────────────────────────────────┘
```

**核心设计解耦**：
1. **阻塞问题解决**：由于 LangGraph 在执行某个具体 Agent 节点（如 `yunyou_agent`）时是同步阻塞的，如果不使用多线程，期间产生的所有底层系统日志（`log.info`）会被闷在内存中，直到节点执行完毕才会一次性发给前端。
2. **多线程并发**：现在我们将 `graph.stream` 的执行过程放入一个独立的后台线程。
3. **全局日志拦截**：系统复写了 `CustomLogger.add_global_sse_callback`，允许主线程捕获所有子线程打印的标准 `logging` 日志，并将其转化为 `thinking` 事件送入线程安全的 `queue.Queue`。
4. **统一分发下发**：主线程维持一个不死的 `while True` 循环，只要队列里有任何东西（无论是模型的 stream token，还是工具调用的中间状态，抑或是网络请求错误），立刻 `yield` 给前端，实现极致丝滑的“打字机”体验。

事件类型说明：

| 事件类型 | 触发场景 |
|---------|------|
| `response_start` | 请求刚到达，准备开始生成 |
| `thinking` | Supervisor 路由决策 / RAG 检索命中 / **所有底层的系统日志 (如 502 网络错误等)** / 工具调用详情 |
| `stream` | AI 逐字输出的文本 / 因执行失败人为构造的（synthetic）最终错误信息提示 |
| `message` | AI 一次性回传的完整消息（主要用于历史消息渲染，流式传输中不再依赖） |
| `interrupt` | 触发人工审批流程（如确认数据库查询） |
| `error` | 请求执行链路崩溃的报错 |
| `response_end` | 后端协程处理完毕，正式关流 |

## 8. 数据库设计

使用 PostgreSQL，连接配置在 `.env`：

```env
POSTGRES_HOST=localhost
POSTGRES_PORT=5432
POSTGRES_USER=postgres
POSTGRES_PASSWORD=your_password
POSTGRES_DB=xf_ai_agent
```

SQLAlchemy 连接池配置（`db/__init__.py`）：

| 参数 | 值 | 说明 |
|------|-----|------|
| `pool_pre_ping` | True | 每次借出连接前检查可用性 |
| `pool_recycle` | 3600 | 1小时回收连接 |
| `pool_size` | 20 | 常驻连接池大小 |
| `max_overflow` | 30 | 突发流量额外连接数 |

## 9. 安全认证机制

使用 **JWT Bearer Token** 认证：

1. 用户登录 → 后端生成 `access_token` + `refresh_token`
2. 前端将 `access_token` 存入 `localStorage.apiKey`
3. 每次 API 请求自动附加 `Authorization: Bearer <token>` 头
4. 后端 `security.py` 通过 FastAPI 的 `Depends(verify_token)` 校验
5. Token 过期 → 前端 `api.js` 检测 401 → 自动跳转登录页

## 10. 本地开发环境搭建

### 后端

```bash
# 1. 安装依赖
cd xf-ai-agent
pip install -e .  # 或 pip install -r requirements.txt

# 2. 配置环境变量
cp .env.example .env
# 编辑 .env 填入数据库、模型服务等配置

# 3. 启动服务
uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload
```

### 前端

```bash
# 1. 安装依赖
cd xf-ai-agent-app
npm install

# 2. 配置环境变量
echo "VITE_API_URL=http://localhost:8000/api/v1" > .env.local

# 3. 启动开发服务器
npm run dev
```

## 12. 扩展指南

### 添加新的 Agent

1. 在 `app/agent/agents/` 创建新文件（如 `finance_agent.py`）
2. 继承 `BaseAgent`，实现 `_build_graph()` 方法
3. 在 `app/agent/registry.py` 中注册
4. Supervisor 会自动发现并路由到新 Agent

### 添加新的模型服务

1. 在 `app/agent/llm/unified_loader.py` 的 `service_router` 字典中添加映射
2. 如需特殊加载逻辑，在 `loader_llm_multi.py` 中实现加载函数
3. 前端通过设置页面即可选择新模型
